# Tópico 1 — Host e Problema

## Papel e duração
- Apresentador responsável por abrir o vídeo e contextualizar o projeto.
- Tempo estimado: **5–7 minutos** para manter o ritmo geral de 20–30 minutos.

## Objetivos principais
- Dar boas-vindas, apresentar o grupo e o propósito do trabalho.
- Explicar o problema identificado no GitHub Mobile atual.
- Introduzir as personas Camila e João, criando empatia com o público.
- Fechar com um gancho que motive os próximos tópicos.

## Roteiro sugerido
1. **Saudação e apresentação do time**
   - Nome da disciplina, professor (opcional) e integrantes.
2. **Objetivo do projeto**
   - "Transformar o GitHub Mobile de um app apenas de leitura em uma ferramenta ativa para criação e gerenciamento em movimento."
3. **Problema observado**
   - Frustração dos usuários: não conseguir criar repositórios nem fazer uploads rápidos.
   - Contraste com a versão web que já oferece essas ações.
4. **Personas**
   - **Camila**: dev full-stack que vive em trânsito e precisa subir hotfix no ônibus.
   - **João**: estudante curioso que quer testar uma branch durante a aula.
5. **Gancho para o restante da apresentação**
   - Ressaltar que o app atual prende o usuário e o nosso objetivo é dar liberdade.
   - Conectar com o próximo tópico: "Como redesenhamos a experiência para devolver esse controle?"

## Recursos visuais recomendados
- Slide com foto/nomes do grupo.
- Tela com o objetivo resumido em bullet points.
- Prints do app atual mostrando a ausência dos botões de criação/upload.
- Cards das personas (foto ilustrativa + dor principal + objetivo).

## Pontos de ênfase
- Falar em tom entusiasmado, como um host de talk show.
- Usar linguagem acessível: explicar termos técnicos rapidamente.
- Garantir que a audiência entenda por que o problema é relevante.

## Check-list final
- [ ] Cronometrar fala para não ultrapassar 7 minutos.
- [ ] Ensaiar transição para o Tópico 2 mencionando o redesign e a metodologia.
- [ ] Compartilhar slides com o time antes da gravação final.
